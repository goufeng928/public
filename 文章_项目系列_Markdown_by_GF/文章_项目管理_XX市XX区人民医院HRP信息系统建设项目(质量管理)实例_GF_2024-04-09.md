# 文章_项目管理_XX市XX区人民医院HRP信息系统建设项目(质量管理)实例_GF_2024-04-09

本文将通过 XX市XX区人民医院HRP信息系统建设项目实例 来论述 `质量管理` 在其中的作用。

## 质量管理定义

**信息系统项目管理师教材定义**:

```txt
质量管理是指确定质量方针、目标和职责，并通过质量体系中的质量规划、质量保证和质量控制以及质量改进来使其实现所有管理职能的全部活动。
```

## 如何进行项目质量管理

**(1) 规划质量管理**:

```txt
规划质量管理:
规划质量管理是识别项目及其可交付成果的质量要求和标准，并准备对策确保符合质量要求的过程。

规划质量管理技术:
成本效益分析、质量成本、实验设计、会议等

规划质量管理输出:
质量管理计划、过程改进计划等
```

**(2) 实施质量保证**:

```txt
实施质量保证:
实施质量保证是审计质量要求和质量控制测量结果，确保采用合理的质量标准和操作性定义的过程。

实施质量保证技术:
质量审计、过程分析等

实施质量保证输出:
变更请求等
```

**(3) 控制质量**:

```txt
控制质量:
质量控制是监督并记录质量活动执行结果，以便评估绩效，并推荐必要的变更过程。

控制质量技术:
统计抽样、检查、质量控制工具等

控制质量输出:
质量控制测量结果、确认的变更等
```

## XX市XX区人民医院HRP信息系统建设项目(质量管理)实例

**项目背景**:

```txt
# 2023 年 9 月, 我公司通过公开招标的方式中标了 XX 市 XX 区人民医院 HRP 信息系统建设项目, 我作为乙方项目经理全程参与了该项目的建设工作。
  本项目合同金额 3,630,000.00 元, 主要建设内容包括基础平台系统、财务会计系统、全面预算管理系统、科室成本核算系统、项目成本核算系统、病种成本核算系统、固定资产管理系统、合同管理系统、人力资源管理系统、统一对账系统、智能发放系统、报销管理系统、移动应用、决策分析(BI)、接口服务的建设。
  建设工期 8 个月, 前 5 个月内完成项目实施建设, 后 2 个月上线试运行, 最终验收 1 个月。
  该项目采用 B/S 架构, 前端展现包括系统入口、总览和数据仪表盘、小程序和移动APP, 后端主要采用Java、C#等语言开发模块, 实现预算编制模块、成本控制模块、资源管理模块、财务分析模块、凭证管理模块、成本核算模块、员工信息模块、薪酬绩效模块、报表与分析模块、集成模块等。
  关键技术指标要求实现系统的用户、权限、引入、输出、日志等功能, 要求支持 EAI 集成, 要求实现 PDM 系统接口、配置、规则等, 要求实现财务会计和预算会计双重功能平行记账、支持新会计制度要求的相关会计报表的生成, 要求实现资产条码管理, 报销单自动分摊、归集等。
  该项目于 2024 年 5 月顺利完工并通过客户验收, 获得了客户的肯定。
  本文以该项目为例, 结合作者实践经验, 探讨信息化系统项目建设过程中在质量管理方面经常遇到的问题及解决方法。
  主要就如何做好规划质量管理、实施质量保证、控制质量这三个方面来进行阐述。最后总结项目中的成功经验, 以及存在的不足之处和具体改进措施。
```

**质量管理论述**:

```txt
# 鉴于该项目对公司意义重大, 公司为此组建了单独的项目团队并任命我为项目经理。
  收到委任后, 我仔细分析了项目合同、工作说明书等相关因素后得出: 本项目范围广、工期紧、运行环境复杂、质量要求高。
  怎样规划质量管理为管理项目质量提供指南和方针; 怎样实施质量保证为项目质量的持续改进创造条件; 怎样控制质量以确保满足项目质量要求，这些都是本项目需要重点考虑的问题。
  因此我认为有效的质量管理是该项目成功的关键。

# 基于质量管理在本项目的重要性, 作为项目经理我在本项目中除了对其它各管理领域恪尽职守外, 还特别对干系人从以下几个方面进行了管理:

# 一、规划质量管理
* 规划质量管理是识别项目及其可交付成果的质量要求和标准，并准备对策确保质量符合要求的过程。
  质量不是每个指标都越优越好，质量越高用于质量管理的成本就会越高; 质量越低项目就可能会陷入无尽的返工中，最终导致进度落后; 质量最关键是要满足我们客户的要求，不在客户要求的范围内的质量是没有意义的。
* 本项目客户的关键质量要求是:
  1. 安全等级保护要达标;
  2. 系统性能要可靠;
  3. 设计要规范;
  4. 要易于扩展;
  5. 用户体验要好。
  理解了客户的要求，接下来就要制订相关的质量测量指标和质量管理流程。
  我和质量工程师黄工在制订质量管理计划时，主要是权衡质量和成本的关系，质量和进度的关系，然后结合客户的要求，以此来制订相关的测量标准和管理策略。
  最后将制订出的计划和相关测量指标与包括客户在内的相关干系人进行评审后，成为了项目质量基准。

# 二、实施质量保证
* 实施质量保证是审计质量要求和质量控制测量结果，确保项目采用了合理的质量标准和操作性定义的过程。
* 为了做好本项目的质量保证工作，我们项目团队主要采取了如下措施来进行管理:
  1. 针对当前系统安全等级保护要求，请专业的信息安全咨询公司来帮助我们做等保评测、差距分析、整改建议，并协助我们进行备案审核。
  2. 针对系统性能要求，我们主要在设计初期就搭建出系统的基础架构并进行性能测试，确保架构符合性能指标后再进行后续工作。
  然后在开发阶段，加强系统测试，发现问题及时解决。在开发完成后，还聘请第三方测试机构进行系统测评，并出具了相关的评测合格证书。
  3. 针对设计规范问题和扩展问题，主要是在设计之初就制订一系列的规范手册并予以执行，然后在设计时充分分析客户的现有系统，尽量做到利旧为主，并在接口上进行规范使之易于对接。
  4. 针对用户体验问题，我们在 beta 版设计了用户反馈窗口，然后在系统线上试运行的三个月里，积累了大量的用户反馈数据。
  我们对这些问题进行分析并对系统进行优化，最终的用户体验非常好。

# 三、控制质量
* 质量控制就是监控项目质量结果以确保其符合既定的质量标准，并根据需要采取相应的变更措施以消除过程低效或质量低劣的原因，确定项目的可交付成果满足主要干系人的既定需求。
* 在线上试运行阶段，某次移动 APP 升级后，我们发现了一个问题。
  部分移动 APP 用户经常会出现无故闪退现象。
  我们对这些用户的系统信息进行收集并分析，发现问题主要集中在某厂家的一个低版本系统里。
  我们进行了模拟测试，发现是我们的 APP 与该系统不兼容所致。
  为了进一步分析，我们找了多个厂家的低版本系统进行测试，发现确实是我们 APP 的问题。
  然后我们进行了版本回溯，发现在旧版本下相应问题消失，直至我们可以肯定是这次的程序升级出现了问题。
  然后我们的技术人员对代码进行了仔细排查，最终修复了该 BUG。

# 经过我们团队历时 8 个月的辛苦努力, 本项目于 2024 年 5 月如期顺利完工并通过客户验收。
  总结整个项目的实施, 我认为本项目的成功得益于我质量管理的有效运用。
  通过结合项目实际特点进行合理规划并制订了一份切实可行的质量管理计划，使得项目质量管理工作能有条不紊的进行; 通过实施多种质量保证措施，使得项目质量持续改进; 通过持续监控项目质量并采取措施进行纠正和预防，使得项目质量总是能满足主要干系人的既定需求。
  通过这一系列方法的实施, 建立了一套高效的质量管理机制, 为项目的成功实现提供了有效保障。
* 当然还是有不足之处, 比如因资料员的粗心，将一张签字后的质量核对单丢失。
  虽然经过补签后解决，但是也花费了不少精力和时间。
  在今后的工作中我将更加注重这方面的管理, 不断的学习充电, 努力提升自身管理水平, 为我国的信息化建设贡献自己的力量。
```
