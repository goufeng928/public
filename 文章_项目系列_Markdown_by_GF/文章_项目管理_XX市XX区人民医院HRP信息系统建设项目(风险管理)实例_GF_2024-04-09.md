# 文章_项目管理_XX市XX区人民医院HRP信息系统建设项目(风险管理)实例_GF_2024-04-09

本文将通过 XX市XX区人民医院HRP信息系统建设项目实例 来论述 `风险管理` 在其中的作用。

## 风险管理定义

**信息系统项目管理师教材定义**:

```txt
项目风险是一种不确定的事件和条件，一旦发生，对项目目标产生某种正面或负面的影响。
项目风险管理的目标在于增加积极事件的概率和影响，降低项目消极事件的概率和影响。
```

## 如何进行项目风险管理

**(1) 规划风险管理**:

```txt
规划风险管理:
风险管理规划指决定如何进行项目风险管理活动的过程。
风险管理过程的规划对保证风险管理 (包括风险管理程度、类型和可见度) 与项目风险程度和项目对组织的重要性相适应起着重要作用，它可保证为风险管理活动提供充足的资源和时间，并确立风险评估一致同意的基础。

规划风险管理的技术:
分析技术、专家判断、会议

规划风险管理的输出:
风险管理计划
```

**(2) 识别风险**:

```txt
识别风险:
识别风险指确定哪些风险会影响项目，并将其特性记载成文。
识别风险是一项反复过程。
随着项目生命期的绩效，新风险可能会出现。

识别风险的方法:
信息收集技术、假设分析、专家判断等

识别风险的输出:
风险登记册
```

**(3) 实施定性风险分析**:

```txt
实施定性风险分析:
定性风险分析指通过考虑风险发生的概率，风险发生后对项目目标的影响和其他因素 (如时间框架和项目 4 大制约条件，即成本、进度、范围和质量) 的风险承受水平，对已识别风险的优先级进行评估。

实施定性风险分析的技术:
风险概率和影响评估、风险紧迫性评估等

实施定性风险分析的输出:
项目文件更新
```

**(4) 实施定量风险分析**:

```txt
实施定量风险分析:
定量风险分析是指对定性风险分析过程中作为对项目需求存在潜在重大影响而排序在先的风险进行分析。
定量风险分析过程是对这些风险事件的影响进行分析，并就风险分配一个数值。
定量风险分析是在不确定情况下进行决策的一种量化方法。

实施定量风险分析的技术:
定量风险分析和建模技术等

实施定量风险分析的输出:
项目文件更新
```

**(5) 规划风险应对**:

```txt
规划风险应对:
规划风险应对指为项目目标增加实现机会，减少失败威胁而制订方案，决定应采取对策的过程。

规划风险应对的技术:
消极、积极、应急风险应对策略、专家判断

规划风险应对的输出:
项目管理计划更新、项目文件更新
```

**(6) 控制风险**:

```txt
控制风险:
在项目生命期实施项目管理计划包含的风险应对措施时，应持续对项目工作进行监督以寻找新风险和变化的风险。

控制风险的技术:
风险再评估、风险审计等

控制风险的输出:
工作绩效信息、变更请求等
```

## XX市XX区人民医院HRP信息系统建设项目(风险管理)实例

**项目背景**:

```txt
# 2023 年 9 月, 我公司通过公开招标的方式中标了 XX 市 XX 区人民医院 HRP 信息系统建设项目, 我作为乙方项目经理全程参与了该项目的建设工作。
  本项目合同金额 3,630,000.00 元, 主要建设内容包括基础平台系统、财务会计系统、全面预算管理系统、科室成本核算系统、项目成本核算系统、病种成本核算系统、固定资产管理系统、合同管理系统、人力资源管理系统、统一对账系统、智能发放系统、报销管理系统、移动应用、决策分析(BI)、接口服务的建设。
  建设工期 8 个月, 前 5 个月内完成项目实施建设, 后 2 个月上线试运行, 最终验收 1 个月。
  该项目采用 B/S 架构, 前端展现包括系统入口、总览和数据仪表盘、小程序和移动APP, 后端主要采用Java、C#等语言开发模块, 实现预算编制模块、成本控制模块、资源管理模块、财务分析模块、凭证管理模块、成本核算模块、员工信息模块、薪酬绩效模块、报表与分析模块、集成模块等。
  关键技术指标要求实现系统的用户、权限、引入、输出、日志等功能, 要求支持 EAI 集成, 要求实现 PDM 系统接口、配置、规则等, 要求实现财务会计和预算会计双重功能平行记账、支持新会计制度要求的相关会计报表的生成, 要求实现资产条码管理, 报销单自动分摊、归集等。
  该项目于 2024 年 5 月顺利完工并通过客户验收, 获得了客户的肯定。
  本文以该项目为例, 结合作者实践经验, 探讨信息化系统项目建设过程中在风险管理方面经常遇到的问题及解决方法。
  主要就如何做好风险管理规划、识别风险、定性风险分析、定量风险分析、规划风险应对、控制风险这六个方面来进行阐述。最后总结项目中的成功经验, 以及存在的不足之处和具体改进措施。
```

**风险管理论述**:

```txt
# 鉴于该项目对公司意义重大, 公司为此组建了单独的项目团队并任命我为项目经理。
  收到委任后, 我仔细分析了项目合同、工作说明书相关因素后得出: 本项目工期紧、运行环境复杂，性能要求高。
  怎样规划风险管理为风险管理活动提供指南和方针; 怎样识别风险为风险管理提供重要依据; 怎样准备风险对策以便及时采取措施进行风险管控，这些都是本项目需要重点考虑的问题。
  因此我认为有效的风险管理是该项目成功的关键。

# 由于风险管理在该项目中的重要性, 作为项目经理我在本项目中除了对其它各管理领域恪尽职守外, 还特别对风险管理从以下几个方面进行了管理:

# 一、规划风险管理
* 规划风险管理是规划和设计如何进行风险管理活动，并制订风险管理计划的过程。
* 在项目启动会时，客户总负责人王院长在会议上对我们的系统提出了 5 点重要要求:
  1. 安全等级保护要符合;
  2. 关键性能指标要达标;
  3. 设计要规范、资料要齐全;
  4. 充分利旧、易于扩展;
  5. 必须在计划内完工上线。
  从这 5 点要求上就为本项目风险管理工作确定了一个大的方向。
  会后我们项目管理团队进行开会讨论，基于客户的要求和项目的实际特点，先把主要的风险项确定出来，然后把风险管控的时间及相关责任人等做了一个概要的安排，以此制订了项目风险管理计划。

# 二、识别风险
* 识别风险是指确定出哪些风险会影响项目，并将其特性记载成文。
  识别风险是一项反复的、需要全员参与的过程。随着项目工作的逐步推进，收集的需求越来越多，风险问题也随之而来，我们将这些问题分成两大类。
  第一是组织外的，主要包括客户需求不确定所带来的风险; 与客户的沟通不畅通所带来的风险; 硬件供应商供货质量及时间上的风险等。
  第二是组织内的，主要包括技术风险、人力风险、质量风险、进度风险、成本风险等。
  我们将这些风险依特性分类整理成册，以便为管理项目风险提供依据。

# 三、实施定性风险分析
* 实施定性风险分析就是对已识别风险出现的概率、产生的影响、风险的紧迫性等进行分析，对风险的优先级进行评估。

# 四、实施定量风险分析
* 在实施定性风险分析后，我们主要对项目中优先级较高的风险进行量化分析。
  通过量化分析可以帮助我们有针对性的开展风险管理工作。

# 五、规划风险应对
* 规划风险应对是为项目目标增加实现机会、减少失败威胁而制订方案和准备对策的过程。
* 对于本项目的重要指标之一安全等级保护问题，我们请了专业的信息安全公司对我们的系统进行评测、分析，并帮助备案审核，这样可以将等保风险转移给他们。
  由于本系统为多用户并发应用系统，系统性能要求较高，所以在设计时就要求先设计和搭建出系统的基础架构并进行性能测试，确保架构符合性能指标后再进行后续工作。
  对于需求风险，我们主要是采用原型图来收集高质量的用户需求并书面确定。
  对于沟通风险，我们主要是在项目之初就与各干系人商量好沟通方式和渠道，在项目进行中保持好与各干系人之间的联系和感情。
  在重要岗位的人员上，我们将核心工作分配给多人，并加强同类型人才的培养和锻炼，同时还准备了后备人员方案。
  对于进度风险，我们主要是增加项目监控力度和频率，加强质量管理以避免返工。
  对于管理风险，我们主要是制订一些列规范和标准，凡事遵循工作核准制度，尽量减少个人主观意识带来的风险。

# 六、控制风险
* 控制风险就是通过持续监控项目风险状态，以识别新风险和控制已识别风险的过程。
  通过对收集到的数据和绩效信息进行分析，对比计划找出它们之间的偏差，并根据需要实施变更和采取预防或纠正措施以控制项目风险。

# 经过我们团队历时 8 个月的辛苦努力, 本项目于 2024 年 5 月如期顺利完工并通过客户验收。
  总结整个项目的实施, 我认为本项目的成功得益于我风险管理的有效运用。
  通过重点规划并制订了一份切实可行的风险管理计划，使得项目风险管理工作有条不紊的进行; 通过反复识别并为风险准备对策，增加了项目实现机会减少了失败威胁; 通过持续监控项目风险状态，使得我们可以采取预防、纠正、补救措施来保证项目风险正常。
  通过这一系列方法的实施, 建立了一套高效的风险管理机制, 为项目的成功实现提供了有效保障。
* 当然还是有不足之处, 比如因为资料员的粗心，将一张核对过的风险单丢失，虽然最后经过补签和重新核对后解决，但是也浪费了不少时间和精力。
  在今后的工作中我将更加注重这方面的管理, 不断的学习充电, 努力提升自身管理水平, 为我国的信息化建设贡献自己的力量。
```
