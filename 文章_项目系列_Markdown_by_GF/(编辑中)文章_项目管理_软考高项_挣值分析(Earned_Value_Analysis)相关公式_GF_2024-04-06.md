# 文章_项目管理_软考高项_挣值分析(Earned_Value_Analysis)相关公式_GF_2024-04-06

相关概念:

```txt
实际成本 (AC, Actual Cost): 指项目已经投入的实际费用 (一般直接给出, 或通过已知条件, 简单计算可得)。

计划值 (PV, Planned Value): 指项目在某个时间点上应该已经完成的工作的预计值。

挣值 (EV, Earned Value): 指项目完成了多少预算的工作 (一般会通过文字, 语言描述出来, 比如: C 活动刚开始, D 活动已进行一半, E 活动已完成。此时只需翻译成进度百分比即可, 牢牢记住 EV = PV * SPI)。

完工预算 (BAC, Budget at Completion): 为将要执行的工作所建立的全部预算的总和。相对于 EAC 来说, 这是旧的总成本。

完工估算 (EAC, Estimate at Completion): 根据项目的绩效和风险量化对项目最可能的总成本所做的一种预测, 即根据截止到目前的工作效率和工作方法, 如果不采取任何措施, 到完工时, 需要花费的成本预算。相对于 BAC 来说, 这是新的总成本。等于截至目前的实际成本加上完工尚需估算。

完工尚需估算 (ETC, Estimate to Completion): 即在某个时间点, 预测完成剩余的工作还需要多少成本。

成本偏差 (CV, Cost Variance): 表示实际成本与挣值之间的差异, 用于衡量项目的成本控制情况 (CPI > 1, CV > 0 成本节约; CPI < 1, CV < 0 成本超支)。

进度偏差 (SV, Schedule Variance): 表示挣值与计划值之间的差异, 用于衡量项目的进度控制情况 (SPI > 1, SV > 0 进度提前; SPI < 1, SV < 0 进度落后)。

成本绩效指数 (CPI, Cost Performance Index): 它衡量正在进行的项目的成本效率。当 CPI 在 1.0 以下或者逐渐变小时, 就应该采取纠正措施。

进度绩效指数 (SPI, Schedule Performance Index): 它衡量正在进行的项目的计划值完成效率。用于评估项目实际进度与计划进度之间的偏差。
```

相关公式:

```txt
* 总预算 = BAC + 管理储备

    * BAC: 成本基准。
    
    * BAC 包含应急储备, 不包含管理储备。
    
    * 计算挣值时, 用的是 BAC。

* 完工尚需估算 ETC 的计算方式:

    [方法 1] 纠偏 (纠偏到 CPI = 1), 按照预算单价计算, 且不考虑进度偏差。
    
        非典型(纠偏): 当前导致发生偏差的原因被视为一种特例, 且团队认为将来不会再发生类似的偏差。即这个偏差不是典型的, 后续工作不会遇到, 这就是非典型。
    
        ETC = 剩余工作的计划值 = 总计划值 - 已完成工作的计划值 = BAC - EV
    
    [方法 2] 按照当前 CPI, 即不纠偏 (现在什么 CPI 就继续保持), 且不考虑进度偏差。
    
        典型(不纠偏): 当前出现的偏差被视为具有典型性, 可以代表未来的偏差, 会一直伴随着未来的工作, 这就是典型。
    
        ETC = 剩余工作的计划值 / 成本绩效指数 = (总计划值 - 已完成工作的计划值) / CPI = (BAC - EV) / CPI
        
        注意, 当不纠偏时, 需要除以 CPI, 可以这样理解:
        
        - 以 CPI = 1 来计算, 则剩余工作的计划值为 BAC – EV, 那么不纠偏, 即 CPI <> 1, 以新的 CPI (不等于1) 来计算, 剩余工作的计划值就是需要除以这个比例, 这就得到了 ETC = (BAC – EV) / CPI。
            
        - 即可以把纠偏看作是不纠偏的特殊情况, 即 CPI = 1 时的情况。

* 成本绩效指数 CPI 的计算方式:

    CPI = EV / AC

* 进度绩效指数 CPI 的计算方式:

    SPI = EV / PV
```
