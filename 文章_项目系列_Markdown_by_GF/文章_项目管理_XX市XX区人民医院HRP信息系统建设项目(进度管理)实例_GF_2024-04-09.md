# 文章_项目管理_XX市XX区人民医院HRP信息系统建设项目(进度管理)实例_GF_2024-04-09

本文将通过 XX市XX区人民医院HRP信息系统建设项目实例 来论述 `进度管理` 在其中的作用。

## 进度管理定义

**信息系统项目管理师教材定义**:

```txt
项目进度管理是指在项目实施过程中, 对各阶段的进展程度和项目最终完成的期限所进行的管理。
是在规定的时间内拟定出合理且经济的进度计划 (包括多级管理的子计划), 在执行该计划的过程中, 经常要检查实际进度是否按计划要求进行。
若出现偏差便要及时找出原因, 采取必要的补救措施或调整、修改原计划, 直至项目完成。
其目的是保证项目能在满足其时间约束条件的前提下实现其总体目标。
```

**壹句话总结**:

```txt
项目进度管理是指在项目实施过程中, 对各阶段的进展程度和项目最终完成的期限所进行的管理。
其目的是保证项目能在满足其时间约束条件的前提下实现其总体目标。
```

## 如何进行项目进度管理

**(1) 规划进度管理**:

```txt
规划进度管理:
规划项目进度管理是为实施项目进度管理制定政策、程序, 并形成文档化的项目进度管理计划的过程。
本过程的主要作用是, 如何在整个项目过程中管理、执行和控制项目进度提供指南和方向。

规划进度管理的技术:
分析技术、专家判断等

规划进度管理的输出:
项目进度管理计划 (项目进度管理计划是项目管理计划的组成部分。为编制、监督和控制项目进度建立准则和明确活动)
```

**(2) 定义活动**:

```txt
定义活动:
定义活动过程就是识别和记录为完成项目可交付成果而需采取的所有活动。
其主要作用是, 将工作包分解为活动, 作为对项目工作进行估算、进度规划、执行、监督和控制的基础。

定义活动的方法:
分解、滚动式规则等

定义活动的输出:
活动清单、活动属性、里程碑清单
```

**(3) 排列活动顺序**:

```txt
排列活动顺序:
排列活动顺序是识别和记录项目活动之间的关系的过程。
本过程的主要作用是, 定义工作之间的逻辑顺序, 以便在既定的所有项目制约因素下获得最高的效率。

排列活动顺序的技术:
确定依赖关系、紧前关系绘图法等

排列活动顺序的输出:
项目进度网络图等
```

**(4) 估算活动资源**:

```txt
估算活动资源:
估算活动资源是估算执行各项活动所需的材料、人员、设备或用品的种类和数量的过程。
本过程的主要作用是, 明确完成活动所需的资源种类、数量和特性, 以便做出更准确的成本和持续时间估算。

估算活动资源的方法:
自下而上估算、备选方案分析等

估算活动资源的输出:
活动资源需求、资源分解结构等
```

**(5) 估算活动持续时间**:

```txt
估算活动持续时间:
估算活动持续时间是根据资源估算的结果, 估算完成单项活动所需工作时段数的过程。
本过程的主要作用是, 确定完成每个活动所需花费的时间量, 为制定进度计划提供主要输入。

估算活动持续时间的方法:
类比估算、参数估算、三点估算等

估算活动持续时间的输出:
活动持续时间估算等
```

**(6) 制定进度计划**:

```txt
制定进度计划:
制定进度计划是分析活动顺序、持续时间, 资源需求和进度制约因素, 创建项目进度模型的过程。
本过程的主要作用是, 把活动、持续时间、资源、资源可用性和逻辑关系代入进度规划工具, 从而形成包含各个项目活动的计划日期的进度模型。

制定进度计划的技术:
关键路劲法、资源优化技术等

制定进度计划的输出:
进度基准、项目进度计划、进度数据、项目日历等
```

**注意**:

```txt
在某些项目 (特别是小项目) 中, 定义活动、排列活动顺序, 估算活动资源、估算活动持续时间, 制定进度计划等过程之间的联系非常密切, 以至于可视为一个过程, 由一个人在较短时间内完成。(P256 - 6.1)
```

**(7) 控制进度**:

```txt
控制进度:
控制进度是监督项目活动状态, 更新项目进展, 管理进度基准变更, 以实现计划的过程。
本过程的主要作用是, 提供发现计划偏离的方法, 从而可以及时采取纠正和预防措施, 以降低风险。

控制进度的技术:
绩效审查、资源优化技术等

控制进度的输出:
工作绩效信息、进度预测、变更请求等
```

## XX市XX区人民医院HRP信息系统建设项目(进度管理)实例

**项目背景**:

```txt
# 2023 年 9 月, 我公司通过公开招标的方式中标了 XX 市 XX 区人民医院 HRP 信息系统建设项目, 我作为乙方项目经理全程参与了该项目的建设工作。
  本项目合同金额 3,630,000.00 元, 主要建设内容包括基础平台系统、财务会计系统、全面预算管理系统、科室成本核算系统、项目成本核算系统、病种成本核算系统、固定资产管理系统、合同管理系统、人力资源管理系统、统一对账系统、智能发放系统、报销管理系统、移动应用、决策分析(BI)、接口服务的建设。
  建设工期 8 个月, 前 5 个月内完成项目实施建设, 后 2 个月上线试运行, 最终验收 1 个月。
  该项目采用 B/S 架构, 前端展现包括系统入口、总览和数据仪表盘、小程序和移动APP, 后端主要采用Java、C#等语言开发模块, 实现预算编制模块、成本控制模块、资源管理模块、财务分析模块、凭证管理模块、成本核算模块、员工信息模块、薪酬绩效模块、报表与分析模块、集成模块等。
  关键技术指标要求实现系统的用户、权限、引入、输出、日志等功能, 要求支持 EAI 集成, 要求实现 PDM 系统接口、配置、规则等, 要求实现财务会计和预算会计双重功能平行记账、支持新会计制度要求的相关会计报表的生成, 要求实现资产条码管理, 报销单自动分摊、归集等。
  该项目于 2024 年 5 月顺利完工并通过客户验收, 获得了客户的肯定。
  本文以该项目为例, 结合作者实践经验, 探讨信息化系统项目建设过程中在进度管理方面经常遇到的问题及解决方法。
  主要就如何做好规划进度管理, 定义活动, 排列活动顺序, 估算活动资源, 估算活动持续时间, 制定进度计划, 控制进度这七个方面来进行阐述。最后总结项目中的成功经验, 以及存在的不足之处和具体改进措施。
```

**进度管理论述**:

```txt
# 鉴于该项目对公司意义重大, 公司为此组建了单独的项目团队并任命我为项目经理。
  收到委任后, 我仔细分析了项目合同、工作说明书后得出: 本项目范围广、工期紧、任务繁多。
  怎样规划进度管理为管理项目提供指南和方针; 怎样制定进度计划以确定最优的进度计划方案; 怎样控制进度以便及时采取纠正和预防措施来降低风险, 这些都是该项目需要重点考虑的问题。
  因此我认为有效的进度管理是该项目成功的关键。

# 由于进度管理在该项目中显得尤为重要, 作为项目经理我在本项目中除了对其它各管理领域恪尽职守外, 还特别对进度管理从以下几个方面进行了管理:

# 一、规划进度管理
* 在这一过程中, 我带领项目团队成员一起开会讨论, 就如何管理、执行和控制项目进度进行了规划。
  我们规定了用于绩效测量的挣值管理技术、制定进度模型的方法和工具, 还对各种计量单位、报告格式进行了统一。
  然后由我制定一个初步的总体进度计划, 并将总进度计划划分为若干个分计划分配给具体的负责人制订。
  通过这一系列的规划, 为本项目进度管理的有效开展提供了良好基础。

# 二、定义活动
* 为了更好的规划项目、估算资源, 我带领团队成员根据进度管理计划和范围基准等将 WBS 中的工作包进行了进一步分解, 制订出了活动清单和里程碑清单。

# 三、排列活动顺序
* 分解出所有活动后, 我根据各活动之间的依赖关系和优先级, 用前导图法对它们进行了合理排序, 以便在所有项目制约因素下获得最高效率。

# 四、估算活动资源
* 排列好活动顺序后, 我们又对完成活动所需要的资源种类、数量进行了估算, 以便更好的估算成本和时间。

# 五、估算活动持续时间
* 估算完活动资源后, 我们又结合活动清单、资源日历等因素对各项活动完成所需时间进行了估算, 出于慎重考虑, 我们一般会对项目中比较复杂或不确定因素较多的工作增加 10% 的应急储备时间。

# 六、制订进度计划
* 正所谓磨刀不误砍材工, 一个项目如果没有一个好的进度计划就不可能如期完成。制订进度计划就是分析活动顺序、持续时间、资源需求和进度制约因素, 创建进度模型的过程。
* 基于前面对各项工作的分析, 我在制订进度计划时先采用关键路径法制作出初步进度计划; 然后运用资源优化、进度压缩等技术进一步优化; 再分析计划中存在的问题更进一步优化; 最后结合风险分析比较各种方案, 最终确定出最优的进度计划方案。
  下面我以甘特图的形式描述本项目概括性进度计划:
  +---------------------+---------+---------+---------+---------+---------+---------+
  |任务\时间(月)        |0.20个月 |1.20个月 |5.00个月 |6.00个月 |8.00个月 |9.00个月 |
  +---------------------+---------+---------+---------+---------+---------+---------+
  |项目启动             |√        |         |         |         |         |         |
  |需求调研与分析       |         |√        |         |         |         |         |
  |系统设计、搭建、开发 |         |         |√        |         |         |         |
  |系统测试部署培训     |         |         |         |√        |         |         |
  |系统试运行           |         |         |         |         |√        |         |
  |项目终验             |         |         |         |         |         |√        |
  +---------------------+---------+---------+---------+---------+---------+---------+

# 七、控制进度
* 控制进度就是将项目实际表现和进度计划进行对比, 收集并测量各项数据加以分析, 及时采取纠正和预防措施降低风险, 确保项目进度正常。
  在项目进行到第 6 个月时, 我发现产品系统培训计划迟迟不能落实。
  原因是由于客户方部分被培训对象近期需要因公外出, 无法集中到一起进行培训。
  经与客户项目负责人进行沟通和协调后, 我们决定采用现场授课和网络会议的方式同步进行。
  我向公司临时借调了直播设备, 并经过一番准备后, 系统培训工作得以顺利完成。
  由于本次培训采用了直播方式, 所以容纳的听课人数更多, 使得客户方普通员工也能直接参与进来, 大大的加快了系统应用在客户方的普及度, 客户对此非常满意。

# 经过我们团队历时 8 个月的辛苦努力, 本项目于 2024 年 5 月如期顺利完工并通过客户验收。
  总结整个项目的实施, 我认为本项目的成功得益于我有效的进度管理。
  通过科学的规划进度管理, 使得项目得以有条不紊的进行; 通过制订了一份切实可行的进度计划, 使得项目得以以最优的方式进行; 通过持续监控进度状态并及时纠偏, 使得项目得以按时完成。
  通过这一系列方法的实施, 建立了一套高效的进度管理机制, 为项目的成功实现提供了有效保障。
* 当然还是有不足之处, 比如进度的优化做的还是不够好, 活动时间的估算还是不够准确。
  在今后的工作中我将更加注重进度的优化工作, 多深入到现场, 注重计划与实际相协调, 不断的学习充电, 努力提升自身管理水平, 为我国的信息化建设贡献自己的力量。
```
