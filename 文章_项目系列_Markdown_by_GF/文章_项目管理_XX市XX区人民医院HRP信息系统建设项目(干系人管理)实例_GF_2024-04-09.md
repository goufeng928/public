本文将通过 XX市XX区人民医院HRP信息系统建设项目实例 来论述 `干系人管理` 在其中的作用。

## 干系人管理定义

**信息系统项目管理师教材定义**:

```txt
项目干系人管理是指对项目干系人需求、希望和期望的识别, 并通过沟通上的管理来满足其需要、解决其问题的过程。
项目干系人管理将会赢得更多人的支持, 从而能够确保项目取得成功。
```

**壹句话总结**:

```txt
项目干系人管理就是通过沟通上的管理来满足项目干系人的需要、解决其问题的过程, 其主要作用就是为了在项目中获得更多来自干系人的参与与支持。
```

## 如何进行项目干系人管理

**(1) 识别干系人**:

```txt
识别干系人:
识别能影响项目决策、活动或结果的个人、群体或组织, 以及被项目决策、活动或者结果影响的个人、群体或者组织, 并分析和记录他们的相关信息 (权益、利益、需求、影响力等) 的过程。

干系人分析技术:
重点管理; 令其满意; 随时告知; 花较少精力

识别干系人的输出:
干系人登记册
```

**(2) 规划干系人管理**:

```txt
规划干系人管理:
基于干系人的需求、利益及对项目成功的潜在影响分析, 制定合适的管理策略, 以有效调动干系人参与整个项目生命周期的过程。

规划干系人管理的分析技术:
关键干系人的所需参与程度和当前参与程度。
干系人变更的范围和影响。
干系人之间相互关系和潜在关系。
项目现阶段的干系人沟通需求。等等。

规划干系人管理的输出:
干系人管理计划
```

**(3) 管理干系人参与**:

```txt
管理干系人参与:
在整个项目生命周期中, 与干系人进行沟通和协作, 以满足他的需求与期望, 解决实际出现的问题, 并促进干系人合理参与项目活动的过程。

管理干系人参与的沟通方法:
在管理干系人参与时, 应该使用在沟通管理计划中确定的针对每个干系人的沟通方法。

管理干系人参与的人际关系技能:
应用人际关系技能来管理干系人的期望

管理干系人参与的管理技能:
应用管理技能来达成项目目标

管理干系人参与的输出:
问题日志、变更请求等
```

**(4) 控制干系人参与**:

```txt
控制干系人参与:
监督项目干系人之间的关系, 调整策略和计划, 以调动干系人参与的过程。本过程的作用是随着项目进展和环境变化, 维持并提升干系人参与活动的效率和效果。

控制干系人参与的输出:
工作绩效信息、变更请求
```

## XX市XX区人民医院HRP信息系统建设项目(干系人管理)实例

**项目背景**:

```txt
# 2023 年 9 月, 我公司通过公开招标的方式中标了 XX 市 XX 区人民医院 HRP 信息系统建设项目, 我作为乙方项目经理全程参与了该项目的建设工作。
  本项目合同金额 3,630,000.00 元, 主要建设内容包括基础平台系统、财务会计系统、全面预算管理系统、科室成本核算系统、项目成本核算系统、病种成本核算系统、固定资产管理系统、合同管理系统、人力资源管理系统、统一对账系统、智能发放系统、报销管理系统、移动应用、决策分析(BI)、接口服务的建设。
  建设工期 8 个月, 前 5 个月内完成项目实施建设, 后 2 个月上线试运行, 最终验收 1 个月。
  该项目采用 B/S 架构, 前端展现包括系统入口、总览和数据仪表盘、小程序和移动APP, 后端主要采用Java、C#等语言开发模块, 实现预算编制模块、成本控制模块、资源管理模块、财务分析模块、凭证管理模块、成本核算模块、员工信息模块、薪酬绩效模块、报表与分析模块、集成模块等。
  关键技术指标要求实现系统的用户、权限、引入、输出、日志等功能, 要求支持 EAI 集成, 要求实现 PDM 系统接口、配置、规则等, 要求实现财务会计和预算会计双重功能平行记账、支持新会计制度要求的相关会计报表的生成, 要求实现资产条码管理, 报销单自动分摊、归集等。
  该项目于 2024 年 5 月顺利完工并通过客户验收, 获得了客户的肯定。
  本文以该项目为例, 结合作者实践经验, 探讨信息化系统项目建设过程中在干系人管理方面经常遇到的问题及解决方法。
  主要就如何做好识别干系人, 规划干系人管理, 管理干系人参与, 控制干系人参与这四个方面来进行阐述。最后总结项目中的成功经验, 以及存在的不足之处和具体改进措施。
```

**干系人管理论述**:

```txt
# 鉴于该项目对公司意义重大, 公司为此组建了单独的项目团队并任命我为项目经理。
  收到委任后, 我仔细分析了项目合同、工作说明书后得出: 本项目范围广、工期紧、干系人众多。
  怎样有效识别干系人并清楚他们的权利和利益; 怎样与干系人进行沟通和协作以满足他们的需求和期望; 怎样监督并调整策略以调用干系人的参与, 这些都是该项目需要重点考虑的问题。
  因此我认为有效的干系人管理是该项目成功的关键。

# 由于干系人管理在该项目中显得尤为重要, 作为项目经理我在本项目中除了对其它各管理领域恪尽职守外, 还特别对干系人从以下几个方面进行了管理:

# 识别干系人就是识别那些能影响项目工作或结果, 以及受项目工作或结果影响的个人和群体, 并分析和记录他们的相关信息的过程。
* 在该项目之初, 我通过向公司总经理和业务副总请教, 让我对该项目关键干系人有了比较详细的的了解。
  然后我通过和相关干系人进行交流并分析相关文件, 对该项目干系人做了全面的识别。
  我运用 "权利/利益 方格" 等方法对这些干系人进行了多维度的分析, 并制作出了干系人登记册。
  下面我以表格的形式列出该项目的 "权利/利益 方格", 并给出每个干系人的具体管理策略。
  +-----------------+----------+-------------+
  |干系人           |权利/利益 |管理策略     |
  +-----------------+----------+-------------+
  |公司总经理       |大/大     |重点管理     |
  |公司业务副总     |大/大     |重点管理     |
  |项目建设团队     |小/大     |随时告知     |
  |公司其它职能部门 |小/小     |花较少的精力 |
  |客户             |大/大     |重点管理     |
  |项目监理         |大/小     |令其满意     |
  |供应商           |小/小     |花较少的精力 |
  +-----------------+----------+-------------+

# 规划干系人管理就是基于干系人的需求, 权利, 利益和对项目的影响, 制定合适的管理策略, 以便有效调动干系人参与项目过程。

# 在干系人管理中, 干系人管理是对人进行管理, 沟通管理是对信息进行管理, 需求管理是对干系人需求进行管理, 从某种意义上来说需求管理是干系人管理的一部分。
  项目经理需要通过与干系人之间的有效沟通和协作, 以满足干系人的需求和期望, 最终确保项目成功实现。
* 比如在向公司领导汇报工作时我喜欢用简洁明了的报告方式, 针对需要领导决策的问题, 我会制订多种方案让领导做选择题而不是问答题, 针对领导们最关心的成本和收益问题, 我会在简要报告的同时附带一份详细清单供领导查阅。
  通过这些不同的沟通方法, 有效的满足领导对整个项目大局上的信息需求。
  我还每周组织包括客户、监理等在内的重要干系人召开通气会, 进行进度交底、技术交底、质量交底。
  通过快速频繁的沟通满足了项目干系人对项目关键信息的需求, 并确保了对项目干系人需求、希望的完全理解。

# 随着项目的进展与环境的变化, 各干系人基于自身的利益需求会对项目工作采取不同的态度, 这就需要我们监督好项目干系人之间的关系, 调整策略和计划以调用干系人的参与。
* 在本项目硬件设备采购活动中, 由于供应商方的管理比较混乱, 我们在跟催订单的时候, 常常一会是这个业务员接待, 一会又是另外一个业务员接待。
  而且其业务员还有很多是新手, 对整个业务方面的流程不怎么熟悉。
  这就导致我们的订单跟踪效果非常不理想, 完全不能预计设备的到货时间, 负责物料跟踪的小王向我反映了这一问题。
  我与供应商的业务经理进行了沟通, 要求其指定一名资深的业务人员作为固定负责人对接我们的采购工作, 且必须及时向我们反馈相关信息。
  经过沟通后, 我们的采购订单跟踪信息得到了及时更新, 方便了我对后期工作的安排和调整。

# 经过我们团队历时 8 个月的辛苦努力, 本项目于 2024 年 5 月如期顺利完工并通过客户验收。
  总结整个项目的实施, 我认为本项目的成功得益于我有效的干系人管理。
  通过有效识别并对干系人进行 权利/利益 分析, 使得我们可以针对不同的干系人制订不同的管理策略; 通过与干系人有效的沟通和协调, 使得项目干系人的需求和期望均得到满足; 通过积极监控干系人状态并采取对策调动干系人参与, 获得了更多来自干系人的支持。
  通过这一系列方法的实施, 建立了一套高效的干系人管理机制, 维持并提升了干系人对项目的参与和支持, 为项目的成功实现提供了有效保障。
* 当然还是有不足之处, 比如在验收过程中, 因为项目助理的失误, 没有及时将一条验收信息发送给客户确认, 导致验收时出现了一点小插曲。
  虽然最后经过沟通解决, 但是也浪费了不少时间和精力。
  在今后的工作中我将更加注重这方面的管理, 不断的学习充电, 努力提升自身管理水平, 为我国的信息化建设贡献自己的力量。
```
