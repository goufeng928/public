# 文章_项目管理_XX市XX区人民医院HRP信息系统建设项目(成本管理)实例_GF_2024-04-09

本文将通过 XX市XX区人民医院HRP信息系统建设项目实例 来论述 `成本管理` 在其中的作用。

## 成本管理定义

**信息系统项目管理师教材定义**:

```txt
项目成本管理工作是在项目实施过程中，通过项目成本管理尽量使项目实际发生的成本控制在预算范围之内。
如果项目建设的成本远远超出批准的投资预算，就很容易造成成本失控。
```

**壹句话总结**:

```txt
成本管理是在项目实施过程中，通过对成本进行管理，使项目实际发生的成本控制在预算范围内。
```

## 如何进行项目成本管理

**(1) 规划成本管理**:

```txt
规划成本管理:
规划成本管理是为规划、管理、花费和控制项目成本而制定政策、程序和文档的过程。
本过程的主要作用是，在整个项目中为如何管理项目成本提供指南和方向。

规划成本管理的技术:
专家判断、分析技术、会议

规划成本管理的输出:
成本管理计划
```

**(2) 估算成本**:

```txt
估算成本:
估算成本是对完成项目所需资金进行近似估算的过程。
本过程的主要作用是，确定完成项目工作所需的成本数额。

估算成本的技术:
专家判断、类比估算、三点估算等

估算成本的输出:
活动成本估算、估算依据、项目文件更新
```

**(3) 制订预算**:

```txt
制订预算:
制订预算是汇总所有单个活动或工作包的估算成本，建立一个经批准的成本基准的过程。
本过程的主要作用是，确定成本基准，可据此监督和控制项目绩效。

制订预算的技术:
成本汇总、储备分析、资金限制平衡、专家判断、历史关系

制订预算的输出:
成本基准、项目资金需求、项目文件更新
```

**(4) 控制成本**:

```txt
控制成本:
控制成本是监督项目状态，以更新项目成本，管理成本基准变更的过程。
本过程的主要作用是发现实际与计划的差异，以便采取纠正措施，降低风险。

控制成本的技术:
挣值管理、预测、完工尚需绩效指数等

控制成本的输出:
工作绩效信息、成本预测、变更请求等
```

## XX市XX区人民医院HRP信息系统建设项目(成本管理)实例

**项目背景**:

```txt
# 2023 年 9 月, 我公司通过公开招标的方式中标了 XX 市 XX 区人民医院 HRP 信息系统建设项目, 我作为乙方项目经理全程参与了该项目的建设工作。
  本项目合同金额 3,630,000.00 元, 主要建设内容包括基础平台系统、财务会计系统、全面预算管理系统、科室成本核算系统、项目成本核算系统、病种成本核算系统、固定资产管理系统、合同管理系统、人力资源管理系统、统一对账系统、智能发放系统、报销管理系统、移动应用、决策分析(BI)、接口服务的建设。
  建设工期 8 个月, 前 5 个月内完成项目实施建设, 后 2 个月上线试运行, 最终验收 1 个月。
  该项目采用 B/S 架构, 前端展现包括系统入口、总览和数据仪表盘、小程序和移动APP, 后端主要采用Java、C#等语言开发模块, 实现预算编制模块、成本控制模块、资源管理模块、财务分析模块、凭证管理模块、成本核算模块、员工信息模块、薪酬绩效模块、报表与分析模块、集成模块等。
  关键技术指标要求实现系统的用户、权限、引入、输出、日志等功能, 要求支持 EAI 集成, 要求实现 PDM 系统接口、配置、规则等, 要求实现财务会计和预算会计双重功能平行记账、支持新会计制度要求的相关会计报表的生成, 要求实现资产条码管理, 报销单自动分摊、归集等。
  该项目于 2024 年 5 月顺利完工并通过客户验收, 获得了客户的肯定。
  本文以该项目为例, 结合作者实践经验, 探讨信息化系统项目建设过程中在成本管理方面经常遇到的问题及解决方法。
  主要就如何做好规划成本管理、估算成本、制订预算、控制成本这四个方面来进行阐述。最后总结项目中的成功经验, 以及存在的不足之处和具体改进措施。
```

**成本管理论述**:

```txt
# 鉴于该项目对公司意义重大, 公司为此组建了单独的项目团队并任命我为项目经理。
  收到委任后, 我仔细分析了项目合同、工作说明书后得出: 本项目范围广、工期紧、需求不明确、预算有限。
  怎样规划成本管理为管理项目成本提供指南和方针; 怎样制订预算以便确定项目成本基准; 怎样控制成本以确保项目在批准的预算内完成, 这些都是该项目需要重点考虑的问题。
  因此我认为有效的成本管理是该项目成功的关键。

# 由于成本管理在该项目中显得尤为重要, 作为项目经理我在本项目中除了对其它各管理领域恪尽职守外, 还特别对成本管理从以下几个方面进行了管理:

# 一、规划成本管理
* 在这一过程中，我带领团队成员一起开会讨论，就如何管理、花费和控制项目成本进行了规划。
  我们规定了成本控制拟用的分析技术和跟踪方法，还对成本的计量单位、精确度、偏差临界值和报告格式等做了统一规定。
  我还以 WBS 为基础，将其中的组件按类分配给了具体的负责人进行详细的成本估算。
  通过这一系列的规划，为项目成本管理的有效开展提供了良好基础。

# 二、估算成本
* 估算成本就是对完成项目活动所需要的资金进行近似估算的过程。
  在这一过程中我首先对成本进行了分类，各工程师主要负责估算其负责部分的可变成本。
  对于固定和间接等成本，我依据公司已有的成本计量规则在估算完成后再进行分摊。
  在估算成本时我们遵循自下而上的估算方法，主要采取下面三个步骤来进行:
      1. 识别并分析成本的构成科目，制作出项目资源矩阵。
      2. 根据已识别的成本构成科目采用专家判断、类比估算等方法估算出成本大小。
      3. 召开会议一起讨论并分析成本估算结果，找出可行的低成本替代方案，尽可能的降低项目估算总成本。

# 三、制订预算
* 制订预算是汇总所有单个活动或工作包的估算成本，建立一个经批准的成本基准的过程。
  其主要作用是，确定成本基准，以此来监督和控制项目绩效。
* 在估算完成本后，我结合范围基准、进度基准等对成本进行了汇总。
  依时间线展开后得出了各阶段初步成本预算。
  我对此预算进行了分析，发现硬件采购项中智能文件柜设备完全可以在项目初验之后再进场，因为在前期的试运行中这个设备暂时还用不上。
  而依合同约定在项目试运行后，甲方会再支付 30% 的款项给我方。
  所以我将这个设备的采购时间向后做了稍微的延期，这样在不影响整体计划的前提下，还能优化项目预算减轻公司资金压力。
* 最后为了应对实施过程中出现的未知的未知事件，确保项目预算的切实可行，我决定设定 10% 的管理储备成本，将管理储备和成本基准相加后得出了项目预算。

# 四、控制成本
* 控制成本就是收集和测量各项数据进行分析以确保项目状态正常，将项目计划成本和实际表现进行对比，发现它们之间的差异，并根据需要采取预防、纠正、补救措施以降低风险。
* 在项目进行到 10% 时，按计划应完成需求调研与分析工作。我运用挣值管理技术对其进行了一次检测。
  对收集的各项数据进行分析后得出，当前工作的 PV = 40万，EV = 35万，AC = 39万，显然当前成本超支了。
  经了解后得知因客户主要业务负责人近期频繁因公外出，造成部分详细需求确认缓慢。
  我和技术负责人讨论后决定采取并行施工的方法，对客户已经确认无误的主要框架和模块先行开发，后续详细需求等客户回来再确认，经调整后进度恢复正常且成本也无超支。

# 经过我们团队历时 8 个月的辛苦努力, 本项目于 2024 年 5 月如期顺利完工并通过客户验收。
  总结整个项目的实施, 我认为本项目的成功得益于我成本管理的成功运用。
  通过科学的规划并制订了一份切实可行的成本管理计划，使得成本管理工作能有条不紊的进行; 通过制订项目成本基准，使得监控项目绩效工作有了依据; 通过持续的监控项目成本状态，使得我们可以及时采取变更措施进行纠偏。
  通过这一系列方法的实施, 建立了一套高效的成本管理机制, 为项目的成功实现提供了有效保障。
* 当然还是有不足之处，比如因过于压缩团建活动经费，导致团建活动效果并不理想。
  经征求总经理意见并获得批准后，我将剩余部分经费以伙食补贴的形式发放给了大家，大家对此都非常满意。
  在今后的工作中我将更加注重这方面的管理, 不断的学习充电, 努力提升自身管理水平, 为我国的信息化建设贡献自己的力量。
```
