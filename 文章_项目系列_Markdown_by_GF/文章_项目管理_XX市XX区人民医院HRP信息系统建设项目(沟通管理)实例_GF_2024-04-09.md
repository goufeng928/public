本文将通过 XX市XX区人民医院HRP信息系统建设项目实例 来论述 `沟通管理` 在其中的作用。

## 沟通管理定义

**信息系统项目管理师教材定义**:

```txt
项目沟通管理是确保及时、正确地产生、收集、分发、储存和最终处理项目信息所需的过程。
项目沟通管理过程揭示了实现成功所需的人员、观点、信息这三项要素之间的一种联络过程。
```

**壹句话总结**:

```txt
项目沟通管理是确保及时、正确地产生、收集、分发、储存和最终处理项目信息所需的过程。
```

## 如何进行项目沟通管理

**(1) 规划沟通管理**:

```txt
规划沟通管理:
规划沟通管理是根据干系人的信息需要和要求及组织的可用资产情况，制定合适的项目沟通方式和计划的过程。
本过程的主要作用是，识别和记录与干系人的最有效率且最有效果的沟通方式。

规划沟通管理的技术:
沟通需求分析、沟通技术、沟通方法等。

规划沟通管理的输出:
沟通管理计划、项目文件更新。
```

**(2) 管理沟通**:

```txt
管理沟通:
管理沟通是根据沟通管理计划，生成、收集、分发、储存、检索及最终处置项目信息的过程。
本过程的主要作用是，促进项目干系人之间实现有效率且有效果的沟通。

沟通管理的技术:
沟通模型、沟通技术、沟通方法等。

沟通管理的输出:
项目沟通、项目文件更新等。
```

**(3) 控制沟通**:

```txt
控制沟通:
控制沟通是在整个项目生命周期中对沟通进行监督和控制的过程，以确保满足项目干系人对信息的需求。
本过程的主要作用是，随时确保所有沟通参与者之间的信息流动的最优化。

控制沟通的技术:
信息管理系统、专家判断、会议。

沟通管理的输出:
工作绩效信息、变更请求等。
```

## XX市XX区人民医院HRP信息系统建设项目(沟通管理)实例

**项目背景**:

```txt
# 2023 年 9 月, 我公司通过公开招标的方式中标了 XX 市 XX 区人民医院 HRP 信息系统建设项目, 我作为乙方项目经理全程参与了该项目的建设工作。
  本项目合同金额 3,630,000.00 元, 主要建设内容包括基础平台系统、财务会计系统、全面预算管理系统、科室成本核算系统、项目成本核算系统、病种成本核算系统、固定资产管理系统、合同管理系统、人力资源管理系统、统一对账系统、智能发放系统、报销管理系统、移动应用、决策分析(BI)、接口服务的建设。
  建设工期 8 个月, 前 5 个月内完成项目实施建设, 后 2 个月上线试运行, 最终验收 1 个月。
  该项目采用 B/S 架构, 前端展现包括系统入口、总览和数据仪表盘、小程序和移动APP, 后端主要采用Java、C#等语言开发模块, 实现预算编制模块、成本控制模块、资源管理模块、财务分析模块、凭证管理模块、成本核算模块、员工信息模块、薪酬绩效模块、报表与分析模块、集成模块等。
  关键技术指标要求实现系统的用户、权限、引入、输出、日志等功能, 要求支持 EAI 集成, 要求实现 PDM 系统接口、配置、规则等, 要求实现财务会计和预算会计双重功能平行记账、支持新会计制度要求的相关会计报表的生成, 要求实现资产条码管理, 报销单自动分摊、归集等。
  该项目于 2024 年 5 月顺利完工并通过客户验收, 获得了客户的肯定。
  本文以该项目为例, 结合作者实践经验, 探讨信息化系统项目建设过程中在沟通管理方面经常遇到的问题及解决方法。
  主要就如何做好规划沟通管理、管理沟通、控制沟通这三个方面来进行阐述。最后总结项目中的成功经验, 以及存在的不足之处和具体改进措施。
```

**沟通管理论述**:

```txt
# 鉴于该项目对公司意义重大, 公司为此组建了单独的项目团队并任命我为项目经理。
  收到委任后, 我仔细分析了项目合同、工作说明书后得出: 本项目范围广、工期紧、干系人众多、沟通难度大。
  怎样规划沟通管理以识别与干系人最有效的沟通方式; 怎样管理沟通以促进干系人之间的有效沟通; 怎样控制沟通以满足干系人对信息的需求，这些都是本项目需要重点考虑的问题。
  因此我认为有效的沟通管理是该项目成功的关键。

# 由于沟通管理在该项目中显得尤为重要, 儿沟通管理的本质又是为满足干系人的需求而使用的各种管理方法和技能。
  作为项目经理，我在本项目中除了对其它各管理领域恪尽职守外, 还特别对沟通管理和干系人管理从以下几个方面进行了管理:

# 一、识别干系人
* 在项目管理中，沟通的主要对象是人，所以在制订各种沟通计划前必须先对项目中的人进行识别。
  识别干系人就是识别那些能影响项目工作或结果，以及受项目工作或结果影响的个人或群体，并分析和记录他们的相关信息的过程。

# 二、规划沟通管理和规划干系人管理
* 规划沟通管理是根据干系人的信息需要，制定一份合适的沟通计划，以便识别和记录与干系人最有效的沟通方式，使得信息以正确的形式在正确的时间提供给正确的受众。
  规划干系人管理就是基于干系人的需求、权利、利益和对项目的影响，制订合适的管理策略，以便有效调动干系人参与项目的过程。
  下面我以表格的形式简要描述本项目的干系人管理计划:
  +-----------+-----+---------+-----+---------+
  |干系人     |参与 |沟通需求 |频率 |管理策略 |
  +-----------+-----+---------+-----+---------+
  |公司总经理 |领导 |整体需求 |一般 |重点管理 |
  |项目团队   |支持 |具体需求 |频繁 |随时告知 |
  |职能部门   |支持 |辅助需求 |少   |较少精力 |
  |客户       |支持 |产品需求 |频繁 |令其满意 |
  |供应商     |支持 |采购需求 |一般 |较少精力 |
  +-----------+-----+---------+-----+---------+

# 三、管理沟通和管理干系人参与
* 在整个项目中，干系人管理是对人进行管理，沟通管理是对信息进行管理。
  管理沟通就是根据沟通管理计划，采取多种沟通方法和技术以促进项目干系人之间的有效沟通。
  为了使沟通更加畅通和有效，在项目管理工作中我们要根据不同的沟通对象采取不同的沟通方法。
* 比如在与管理层或客户的研讨会上我大部分会采用 PPT 来叙述，从项目的整体架构、整体流程、重要阶段成果、每个阶段需要投入的人力和资源，以及实现的日期和实现的效果等方面来描述，满足他们从大局观上对项目信息的需求。
  在项目组的日常工作中我们会更加关注某一事物或技术的具体实现，这时候我就会借助更多专业性工具，比如程序流图、数据流图、类图等进行详细分析。
  我还组建了项目微信群，在微信群发布一些及时性通知和信息、展开小规模的讨论、安排一些临时性工作。
  在团队成员发生冲突时我会与他们面对面的深入交谈，倾听他们的述求，解决他们的矛盾。

# 四、控制沟通和控制干系人参与
* 随着项目的进展与环境的变化，各干系人给予其自身的利益需求会对项目工作采取不同的态度。
  这就需要我们监督好项目干系人之间的关系，采取合适的沟通方法，确保干系人对信息的需求，并调整策略和计划以调用干系人的参与。
* 在本项目中期，公司承建了另外一个项目，由于资源有限，需要从本项目组抽调人手。
  但是本项目组人手也只是堪堪够用，而且客户还催的比较紧。
  我向上级领导表明了我的担忧，还向客户委婉表达了我这边的压力。
  经过客户上级领导与我方领导适当沟通后，本项目组人手得以保留，这也使得本项目最终能够如期顺利完成。

# 经过我们团队历时 8 个月的辛苦努力, 本项目于 2024 年 5 月如期顺利完工并通过客户验收。
  总结整个项目的实施, 我认为本项目的成功得益于我沟通管理和干系人管理的有效运用。
  通过科学的规划沟通管理和干系人管理，使得项目中人的信息交流得到有效管理; 通过针对不同的干系人采用不同的沟通方法和技巧，使得各干系人之间的沟通变得非常高效; 通过对项目中人的信息需求进行沟通上的监督和控制，使得各干系人对信息的需求总是能得到及时满足。
  通过这一系列方法的实施, 建立了一套高效的干系人沟通管理机制, 为项目的成功实现提供了有效保障。
* 当然还是有不足之处，比如在验收过程中，因为项目助理没有提前与客户沟通好，导致验收时出现了一点小插曲。
  最后向客户详细说明了情况，最终得到了客户的认可。
  在今后的工作中我将更加注重这方面的管理, 不断的学习充电, 努力提升自身管理水平, 为我国的信息化建设贡献自己的力量。
```
