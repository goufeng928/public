# 文章_项目管理_XX市XX区人民医院HRP信息系统建设项目(整体管理)实例_GF_2024-04-09

本文将通过 XX市XX区人民医院HRP信息系统建设项目实例 来论述 `整体管理` 在其中的作用。

## 整体管理定义

**信息系统项目管理师教材定义**:

```txt
整体管理的基本任务就是按照实施组织确定的程序实现项目目标，将项目管理过程组中需要的各个过程有效形成整体。
```

## 如何进行项目整体管理

**(1) 制定项目章程**:

```txt
制定项目章程:
制定项目章程是制定一份正式批准项目或阶段的文件; 并记录能反映干系人需要和期望的初步要求的过程。
它在项目执行组织与发起组织 (或客户如果是外部项目的话) 之间建立起伙伴关系。
项目章程的批准，标志着项目的正式启动。

制定项目章程的技术:
专家判断、引导技术

制定项目章程的输出:
项目章程
```

**(2) 制定项目管理计划**:

```txt
制定项目管理计划:
制订项目管理计划过程是确定、编制所有部分计划并将其综合和协调为项目管理计划所必需的过程。
项目管理计划是有关项目如何计划、执行、监控及结束的基本信息来源。

制定项目管理计划的技术:
专家判断、引导技术

制定项目管理计划的输出:
项目管理计划
```

**(3) 指导与管理项目工作**:

```txt
指导与管理项目工作:
指导和管理项目执行就是项目经理和管理团队采取多种行动来执行项目管理计划，完成项目范围说明书中的明确工作。

指导与管理项目工作的技术:
专家判断、项目管理信息系统、会议

指导与管理项目工作的输出:
可交付成果、工作绩效数据、变更请求等
```

**(4) 监控项目工作**:

```txt
监控项目工作:
监视和控制项目工作过程是监视和控制启动、规划、执行和结束项目所需的各个过程。
采取纠正或预防措施控制项目的实施效果。
监视是贯穿项目始终的项目管理的一个方面。
监视包括收集、测量并散发绩效信息，并评价测量结果和实施过程改进的趋势。
连续的监视使项目管理团队能够洞察项目的状态是否正常，并识别任何可能要求给予特别注意的方面。

监控项目工作的技术:
专家判断、项目管理信息系统、分析技术、会议

监控项目工作的输出:
变更请求、工作绩效报告等
```

**(5) 实施整体变更控制**:

```txt
实施整体变更控制:
整体变更控制过程贯穿于项目的始终。
由于项目很少会准确地按照项目管理计划进行，因而变更控制必不可少。
项目管理计划、项目范围说明书，以及其他可交付成果必需通过不断地认真管理变更才能得以维持。
否决或批准变更请求应保证将得到批准的变更反映到基准之中。

实施整体变更控制的技术:
专家判断、会议、变更控制工具

实施整体变更控制的输出:
批准的变更请求、变更日志等
```

**(6) 结束项目或阶段**:

```txt
结束项目或阶段:
结束项目或阶段是完结所有项目管理过程组的所有活动，以正式结束项目或阶段的过程。
本过程的主要作用是，总结经验教训，正式结束项目工作，为开展新工作而释放组织资源。

结束项目或阶段的技术:
专家判断、分析技术、会议

结束项目或阶段的输出:
最终产品服务或成果、组织过程资产更新
```

## XX市XX区人民医院HRP信息系统建设项目(整体管理)实例

**项目背景**:

```txt
# 2023 年 9 月, 我公司通过公开招标的方式中标了 XX 市 XX 区人民医院 HRP 信息系统建设项目, 我作为乙方项目经理全程参与了该项目的建设工作。
  本项目合同金额 3,630,000.00 元, 主要建设内容包括基础平台系统、财务会计系统、全面预算管理系统、科室成本核算系统、项目成本核算系统、病种成本核算系统、固定资产管理系统、合同管理系统、人力资源管理系统、统一对账系统、智能发放系统、报销管理系统、移动应用、决策分析(BI)、接口服务的建设。
  建设工期 8 个月, 前 5 个月内完成项目实施建设, 后 2 个月上线试运行, 最终验收 1 个月。
  该项目采用 B/S 架构, 前端展现包括系统入口、总览和数据仪表盘、小程序和移动APP, 后端主要采用Java、C#等语言开发模块, 实现预算编制模块、成本控制模块、资源管理模块、财务分析模块、凭证管理模块、成本核算模块、员工信息模块、薪酬绩效模块、报表与分析模块、集成模块等。
  关键技术指标要求实现系统的用户、权限、引入、输出、日志等功能, 要求支持 EAI 集成, 要求实现 PDM 系统接口、配置、规则等, 要求实现财务会计和预算会计双重功能平行记账、支持新会计制度要求的相关会计报表的生成, 要求实现资产条码管理, 报销单自动分摊、归集等。
  该项目于 2024 年 5 月顺利完工并通过客户验收, 获得了客户的肯定。
  本文以该项目为例, 结合作者实践经验, 探讨信息化系统项目建设过程中在整体管理方面经常遇到的问题及解决方法。
  主要就如何做好制订项目章程、制订项目管理计划、指导和管理项目工作、监控项目工作、实施整体变更控制、结束项目或阶段这六个方面来进行阐述。最后总结项目中的成功经验, 以及存在的不足之处和具体改进措施。
```

**整体管理论述**:

```txt
# 鉴于该项目对公司意义重大, 公司为此组建了单独的项目团队并任命我为项目经理。
  收到委任后, 我仔细分析了项目合同、工作说明书后得出: 本项目范围广、工期紧、干系人众多、各个要素间相互依赖程度较高。
  怎样规划项目管理为项目整体管理工作提供指南和方针; 怎样管理和监控项目工作确保项目按计划进行; 怎样控制和协调好项目中相互竞争又相互依赖的各个过程，这些都是本项目需要重点考虑的问题。
  因此我认为有效的整体管理是该项目成功的关键。

# 由于整体管理在该项目中显得尤为重要, 作为项目经理我在本项目中除了对其它各管理领域恪尽职守外, 还特别对干系人从以下几个方面进行了管理:

# 一、制订项目章程
* 项目章程是一份正式批准项目或阶段的文件，也是项目或阶段启动和执行的重要依据。
  本项目项目章程由 XX 区人民医院王院长亲自签发并制定我为项目经理。
* 收到授权后，我对项目启动会议做了精心的准备和安排。
  邀请到了 XX 区人民医院王院长、具体负责人彭科长，本公司副总和相关干系人出席会议。
  在项目启动会议上，我们除了对相关议题进行讨论外，我还借助于有双方高层领导的背书，在会议上将项目中可能出现的权职问题、流程问题、沟通问题等关键问题一一落实到位，为之后项目工作的有效开展奠定了良好基础。
  会后我们还举行了联欢会，增进了各干系人之间的联系和感情。

# 二、制订项目管理计划
* 在这一过程中，我带领团队班子成员对项目章程、项目合同、工作说明书等进行了仔细分析。
  借助 MS Project 工具，依公司现有程序和模板制作出了初步的项目管理计划。
  然后我们进行分工，由各事项负责人负责编写详细的分计划，然后我运用整体管理的方法进行综合与协调，最后制订出比较详细的项目管理计划。

# 三、指导和管理项目工作
* 在本项目中，我主要采取如下措施确保项目工作按计划来执行。
* 1. 在项目启动后，我组织项目组全体成员进行了一次动员会，为整个项目团队统一了思想、鼓舞了士气。2. 每周我会组织团队班子开会总结上周的工作绩效、出现的问题和未完成的事务等。
  2. 每周我会组织团队班子开会总结上周的工作绩效、出现的问题和未完成的事务等。
  并对各项工作进行整体协调，安排本周的工作任务，并对重要事项和关键指标等进行重点强调。
  3. 定期组织包括甲方在内的相关干系人召开通气会，进行技术交底、进度交底和质量交底。

# 四、监控项目工作
* 监控项目工作就是收集和测量各项数据进行分析，将项目计划和实际表现进行对比，确保项目工作状态正常。
* 在项目进行到 10% 时，按计划应完成需求调研与分析工作。我运用挣值管理技术对其进行了一次检测。
  对收集的各项数据进行分析后得出， 当前工作的 PV = 40万，EV = 35万，AC = 40万，显然当前进度落后、成本超支了。
  经了解后得知因客户主要业务负责人近期因公频繁外出，造成部分详细需求确认缓慢。
  我和技术负责人讨论后决定采取并行施工的方法，对客户已经确认无误的主要框架和模块先行开发，后续详细需求等客户回来再确认，经调整后进度恢复正常且成本也无超支。

# 五、实施整体变更控制
* 由于项目很少会准确的按计划进行，因此变更控制必不可少。
  变更不可怕，可怕的是不经批准不遵守流程的变更。
* 在系统线上试运行时，客户发现移动 APP 端的用户体验不够好，原先设计的业务流程不够优化，需要增加和减少部分功能，于是提出了变更申请。
  我收到申请后对变更内容进行了评估，并和相关干系人一起进行了详细讨论，然后向客户回复了变更方案以及变更费用估算。
  客户同意后，我再将变更申请提交给 CCB 评审。得到 CCB 批准后，我再通知安排执行。
  由于涉及基线变更，我们还对相关配置项进行了变更处理。
  最后我还对本次变更进行了监控和验证，确保变更效果符合我们的变更目的。

# 六、结束项目或阶段
* 在本阶段，我主要是审查项目或阶段工作是否均已完成，并向客户移交产品或服务，总结经验教训，正式结束项目或阶段。

# 经过我们团队历时 8 个月的辛苦努力, 本项目于 2024 年 5 月如期顺利完工并通过客户验收。
  总结整个项目的实施, 我认为本项目的成功得益于我有效的整体管理的成功运用。
  通过科学的规划并制订了一份切实可行的项目管理计划，使得整体管理工作有条不紊的进行; 通过管理和监督项目工作的执行，确保了项目按计划完成; 通过严格的变更控制流程，使得项目在各项关键指标上均能得到有效控制。
  通过这一系列方法的实施, 建立了一套高效的整体管理机制, 将项目管理中的各个过程有效形成整体，为项目的成功实现提供了有效保障。
* 当然还是有不足之处, 比如项目初期对项目的监控还是不够给力，之后我除了要求各组长必须准时汇报工作状态外，还深入到一线进行详细了解。
  在今后的工作中我将更加注重这方面的管理, 不断的学习充电, 努力提升自身管理水平, 为我国的信息化建设贡献自己的力量。
```
